## 四、初始化管理员账号

集合创建完成后，需要手动创建一个老板账号（后续由老板通过小程序创建其他员工）：

**方式一：通过控制台直接插入**

在 `employee` 集合中手动插入一条记录：

```json
{
  "_id": "会自动生成",
  "name": "老板",
  "account": "admin",
  "password": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855",
  // 以上是密码 "admin" 的 SHA256 哈希值，实际使用时替换为真实密码的哈希
  "role": "老板",
  "status": 1,
  "created_at": {
    "$date": "2026-06-12T00:00:00.000Z"
  }
}
```

> **重要**：首次部署后请立即通过控制台修改密码，或通过员工管理页面重新设置。

## 五、初始化系统选项

在 `system_option` 集合中插入初始选项数据：

```json
// 尺码
{ "type": "size", "name": "S", "creator_id": "", "created_at": {} }
{ "type": "size", "name": "M", "creator_id": "", "created_at": {} }
{ "type": "size", "name": "L", "creator_id": "", "created_at": {} }
{ "type": "size", "name": "XL", "creator_id": "", "created_at": {} }
{ "type": "size", "name": "XXL", "creator_id": "", "created_at": {} }

// 性别
{ "type": "size", "name": "男", "creator_id": "", "created_at": {} }
{ "type": "size", "name": "女", "creator_id": "", "created_at": {} }
{ "type": "size", "name": "男女同款", "creator_id": "", "created_at": {} }

// 款式（示例）
{ "type": "style", "name": "夏装", "creator_id": "", "created_at": {} }
{ "type": "style", "name": "冬装", "creator_id": "", "created_at": {} }

// 目的地（示例）
{ "type": "destination", "name": "本校仓库", "creator_id": "", "created_at": {} }
{ "type": "destination", "name": "客户自提", "creator_id": "", "created_at": {} }
```

> 注意：性别选项在 PRD 中说明"仅查看，不可修改"，因此性别选项建议直接在代码中写死或初始化后锁定。

## 六、注意事项

1. **云开发权限**：所有集合建议设置"仅创建者及管理员可读写"，不要在小程序端暴露读写权限
2. **密码安全**：手动插入管理员账号时，使用 SHA256 哈希后的密码，不要明文存储
3. **环境ID**：部署前请在 `app.js` 中替换为实际的环境ID
4. **免费配额**：云开发基础版每日约 1000 次云函数调用，超出后需升级套餐
