## 1. 产品概述

### 1.1 产品背景

随着校服生产规模扩大，企业内部原材料采购、裁剪加工、车间生产、成品出库等环节涉及多个角色、多名员工协同。目前信息流转依靠线下记录或零散工具，存在数据不互通、订单状态不透明、责任归属不清等问题，亟需一套统一的数字化管理平台。

### 1.2 产品目标

搭建一套面向校服生产全流程的微信小程序管理系统，实现：

- **信息透明化**：所有角色实时掌握订单流转状态
- **流程规范化**：出入库、加工、确认等环节线上留痕
- **数据可视化**：老板可实时查看各模块经营数据
- **协作高效化**：减少跨部门沟通成本，提升整体生产效率

### 1.3 目标用户特征

| 用户群体 | 人数规模 | 特征 |
|----------|----------|------|
| 老板 | 多人（2+） | 需要数据总览全局，界面简洁不需复杂操作 |
| 原材料管理员 | 多人 | 负责物料出入库，需快速录入和查询 |
| 裁剪管理员 | 多人 | 接收物料并执行裁剪，需清晰的任务列表 |
| 车间管理员 | 多人 | 执行加工和辅料使用，需与裁剪、成品模块衔接 |
| 成品管理员 | 多人 | 管理成衣库存和出库，需精确的分类统计 |

> **界面设计原则**：鉴于员工年龄多在40岁以上，界面应保持**大字体、高对比度、扁平层级、操作步骤少**。避免复杂嵌套，核心操作一步直达。

## 3. 全局功能约定

### 3.1 通用功能

- **登录方式**：账号 + 密码登录（员工账号由老板在后台创建分发）
- **底部导航**：各角色登录后仅显示其有权限的模块入口
- **消息通知**：各待办事项通过小程序内通知红点提醒
- **搜索**：所有列表页支持按订单号、学校、供应商等关键词搜索
- **筛选**：列表页支持按时间范围、状态等维度筛选
- **图片上传**：支持拍照和相册选择，单张/多张可选；图片压缩后上传
- **备注**：所有"选填"字段在界面上以"选填"标签明示
- **删除操作**：删除前需二次确认弹窗

### 3.2 通用字段

| 字段名 | 说明 |
|--------|------|
| 创建时间 | 自动记录，精确到秒（北京时间） |
| 创建人 | 自动填充当前登录用户 |
| 订单编号 | 系统自动生成，格式：`模块前缀-YYYYMMDD-HHMMSS` |
| 状态 | 每条记录含状态字段，状态流转见各模块说明 |

### 3.3 下拉选项的动态管理

以下字段的选项支持员工自主添加，添加后全局生效（同一角色均可见）：

- 供应商名称
- 原材料二级分类（布料/辅料各自独立）
- 学校名称
- 尺码
- 性别（男/女/男女同款）
- 款式
- 目标车间
- 出库目的地

> 老板可在「系统设置 → 选项管理」中统一管理（增/删）以上选项。

## 4. 模块详细需求

---

### 4.1 原材料管理模块

**适用角色**：原材料管理员（入库+出库）、裁剪管理员（入库+出库）、车间管理员（入库+出库）
**模块入口**：底部 Tab「原材料」

> **库存说明**：所有角色的入库操作均写入**同一份原材料总库存**，各角色共享库存数据。出库时系统校验总库存余量。

#### 4.1.1 原材料入库

**功能描述**：将采购的布料/辅料登记入库。

**操作步骤**：

1. 点击右上角「+ 新增入库」
2. 选择供应商（下拉选择，若无则点击「新增供应商」）
   - **新增供应商弹窗**：填写「供应商名称」「联系人姓名」「联系电话」→ 确认添加
3. 添加物料明细（可添加多条）：
   - **物料一级分类**：布料 / 辅料（下拉，选择后不可修改该条）
   - **物料二级分类**：下拉选择（跟随一级分类联动，若无则点击「新增二级分类」输入名称后使用）
   - **数量**：正整数
4. 上传照片（选填，最多9张）
5. 填写备注（选填，最多200字）
6. 点击「提交入库」

**数据写入门店**：入库后对应物料数量累加到原材料仓库总库存。

**权限说明**：原材料管理员、裁剪管理员、车间管理员均可执行入库操作，出库操作同样由以上三类角色执行。

---

#### 4.1.2 原材料出库

**功能描述**：将原材料出库并指派给裁剪管理员或车间管理员。

**操作步骤**：

1. 点击右上角「+ 新增出库」
2. 添加出库物料（可添加多条）：
   - **物料一级分类**：布料 / 辅料（下拉）
   - **物料二级分类**：下拉选择（联动）
   - **数量**：正整数（系统校验不得超出当前库存）
3. 选择目标角色（下拉选择，支持选择裁剪管理员或车间管理员）
4. 上传照片（选填）
5. 填写备注（选填）
6. 点击「提交出库」

**数据影响**：提交后，目标管理员的库存中对应物料数量增加（裁剪管理员→进入裁剪流程；车间管理员→进入车间辅料库存）。

**权限说明**：原材料管理员、裁剪管理员、车间管理员均可操作。

---

#### 4.1.3 出入库明细

**功能描述**：展示所有入库单和出库单的汇总列表。

**页面布局**：

- 顶部 Tab 切换：「全部 / 入库记录 / 出库记录」
- 每条记录卡片展示：
  - 订单编号、供应商/目标裁剪管理员、状态、时间
  - 点击卡片展开明细（物料名称+数量+照片）
- 支持按时间、关键词搜索

**订单状态**：

| 状态 | 说明 |
|------|------|
| 已完成 | 正常流转完毕 |
| 已取消 | 出库/入库被撤回 |

---

### 4.2 裁剪管理模块

**适用角色**：裁剪管理员
**模块入口**：底部 Tab「裁剪」

#### 4.2.1 待裁剪确认

**功能描述**：裁剪管理员接收来自原材料管理模块的出库物料订单，确认物料是否正确。

**页面布局**：

- 顶部 Tab：「待确认 / 已确认」
- **待确认 Tab**：
  - 列表展示所有待确认的出库单
  - 每条记录：订单编号、来源操作员姓名、物料明细（名称+数量）、时间
  - 操作按钮：
    - **「确认」**：物料无误，确认接单 → 订单移至「已确认」
    - **「有问题」**：物料有误，点击后填写问题描述（必填）→ 通知原材料管理员核查（需二次确认）
  - 点击记录卡片可查看完整详情（含照片、备注）
- **已确认 Tab**：
  - 仅展示已确认的历史记录，只读，不可操作

---

#### 4.2.2 裁剪加工

**功能描述**：裁剪管理员对已确认的来料进行裁剪，填写实际使用量并制定生产计划。

**操作步骤**：

1. 点击右上角「+ 新增裁剪单」
2. 选择已确认的来料订单（下拉，仅显示「已确认」的待裁剪订单）
3. 系统自动填充该订单的物料明细
4. 填写布料/辅料实际使用数量（每项必填）
5. 填写裁剪后的计划衣服信息：
   - **尺码**：下拉选择（可多选）
   - **性别**：下拉选择（男 / 女 / 男女同款）
   - **款式**：下拉选择
   - **学校**：下拉选择
   - **计划件数**：按各尺码分别填写（整数）
6. 选择目标车间（下拉选择）
7. 填写备注（选填）
8. 点击「提交」

**数据影响**：提交后，系统向目标车间管理员推送待加工确认通知。

---

#### 4.2.3 裁剪记录

**功能描述**：展示所有已提交的裁剪单详情。

**页面布局**：

- 列表页：订单编号、目标车间、提交时间、状态
- 点击进入详情页：展示完整裁剪信息（含物料使用量、计划件数各参数、照片、备注）

---

### 4.3 车间管理模块

**适用角色**：车间管理员
**模块入口**：底部 Tab「车间」

#### 4.3.1 待辅料确认

**功能描述**：车间管理员接收来自原材料管理模块的出库物料订单，确认物料是否正确。

**页面布局**：

- 顶部 Tab：「待确认 / 已确认」
- **待确认 Tab**：
  - 列表展示所有待确认的出库单
  - 每条记录：订单编号、来源操作员姓名、物料明细（名称+数量）、时间
  - 操作按钮：
    - **「确认」**：物料无误，确认入库 → 进入辅料库存
    - **「有问题」**：物料有误，点击后填写问题描述（必填）→ 通知原材料管理员核查
- **已确认 Tab**：
  - 仅展示已确认的历史记录，只读，不可操作

---

#### 4.3.2 待加工确认

**功能描述**：车间管理员接收来自裁剪管理员的裁剪单，需核对参数无误后确认。

**页面布局**：

- 列表展示所有待确认的裁剪单
- 每条记录：订单编号、来源裁剪管理员姓名、计划件数、尺码、性别、款式、学校、时间
- 操作按钮：
  - **「确认」**：参数无误，确认接单 → 订单进入「加工登记」可用列表
  - **「退回」**：参数有误，点击后填写退回原因（必填）→ 订单退回至裁剪模块并通知对应裁剪管理员

---

#### 4.3.2 加工登记

**功能描述**：对已确认的裁剪单进行实际加工数据登记。

**操作步骤**：

1. 点击右上角「+ 新增加工单」
2. 选择裁剪单（下拉，仅显示「已确认」且未登记的裁剪单）
3. 系统自动填充：计划件数、尺码、性别、款式、学校（不可编辑）
4. 填写实际加工件数（按各尺码填写整数）
   - 系统自动计算损耗率：`损耗率 = (计划件数 - 实际件数) / 计划件数 × 100%`，保留2位小数
5. 填写辅料使用量（按辅料二级分类分别填写用量）
6. 点击「提交」

**数据影响**：提交后，该加工单自动流转至成品管理员的「车间加工订单确认」列表。

---

#### 4.3.3 加工记录

**功能描述**：展示所有加工登记记录的详细信息。

**页面布局**：

- 列表页：订单编号、来源裁剪单号、目标工厂、提交时间、状态
- 点击进入详情页：展示完整加工信息（含计划件数、实际件数、损耗率、辅料使用量等）

---

### 4.4 成品管理模块

**适用角色**：成品管理员
**模块入口**：底部 Tab「成品」

#### 4.4.1 车间加工订单确认

**功能描述**：成品管理员审核从车间传来的加工单，确认数据无误后入库。

**页面布局**：

- 列表展示所有待确认的加工单
- 每条记录：订单编号、来源车间管理员姓名、实际件数、尺码、性别、款式、学校、时间
- 操作按钮：
  - **「确认」**：数据无误，确认入库 → 成衣进入成品库存
  - **「有问题」**：点击后填写问题描述（必填）→ 通知对应车间管理员

**入库规则**：确认后，成衣按「性别+款式+学校+尺码」四个维度分类入库，库存数量累加。

---

#### 4.4.2 库存明细

**功能描述**：展示所有成衣的库存情况，支持多维度统计。

**页面布局**：

- **筛选栏**：按性别 / 款式 / 学校 / 尺码多维筛选（筛选项可多选）
- **库存统计卡片**：
  - 每个组合（性别+款式+学校+尺码）显示一行数据：`[性别] [款式] [学校] [尺码] — N件`
- **汇总统计栏**（页面底部固定）：
  - 总库存件数、总SKU数
- **各车间数据**：
  - 可按车间名称筛选，查看该车间的产出明细
- 支持导出数据（Excel格式，供老板使用）

---

#### 4.4.3 出库

**功能描述**：将成品出库至指定目的地。

**操作步骤**：

1. 点击右上角「+ 新增出库」
2. 选择加工订单（下拉）
3. 系统自动填充：尺码、性别、款式、学校（不可编辑）
4. 显示当前库存可用数量
5. 填写出库数量（不可超过库存）
6. 选择/新增出库目的地（下拉选择，或点击「新增目的地」输入名称）
7. 点击「确认出库」

**数据影响**：出库后对应库存数量扣减。

---

#### 4.4.4 出库记录

**功能描述**：展示所有出库订单的记录。

**页面布局**：

- 列表页：订单编号、出库目的地、尺码+性别+款式+学校、出库数量、时间
- 支持按目的地、时间范围筛选
- 点击可查看详情（含原始加工单信息）

---

### 4.5 老板模块

**适用角色**：企业老板（多老板账号）
**模块入口**：底部 Tab「老板」（老板专属Tab）

#### 4.5.1 数据总览

**功能描述**：老板登录后默认首页，实时展示各模块关键数据指标。

**页面布局**：

采用卡片式布局，大数字+趋势箭头展示核心指标：

| 指标 | 数据来源 |
|------|----------|
| 今日入库（原材料） | 原材料管理模块 |
| 今日出库（原材料） | 原材料管理模块 |
| 待确认裁剪单 | 裁剪管理模块 |
| 待确认加工单 | 车间管理模块 |
| 待确认成品单 | 成品管理模块 |
| 当前原材料库存（总量） | 原材料管理模块 |
| 当前成品库存（总量） | 成品管理模块 |
| 本月出库总量 | 成品管理模块 |

- 底部可快速跳转查看各模块详细数据
- 数据实时刷新（下拉刷新）

#### 4.5.2 员工管理

**功能描述**：老板添加、删除员工账号，并分配角色。

**操作步骤**：

1. **新增员工**：
   - 点击「+ 新增员工」
   - 填写：姓名、手机号、登录密码、分配角色（单选）
   - 点击「确认添加」
2. **删除员工**：
   - 在员工列表点击某员工右侧「删除」按钮
   - 二次确认后删除（删除后该账号无法登录）
3. **编辑员工**：点击员工卡片可修改姓名、账号、角色
4. 列表展示：姓名、账号、角色、入职日期、状态（正常/已禁用）

> 老板账号本身不可删除。

#### 4.5.3 订单记录

**功能描述**：查看所有订单的完整流转路径和详细信息。

**页面布局**：

- 顶部筛选：按模块（原材料/裁剪/车间/成品）、按状态、按时间
- 每条订单记录展示：
  - 订单编号、当前状态
  - **流转时间轴**：按时间顺序展示该订单经过的所有节点（创建→各环节确认→完成），点击每个节点可查看该节点详情
  - 关联人员：显示每一步的操作人
- 支持关键词搜索

#### 4.5.4 系统设置

**功能描述**：老板统一管理系统选项配置。

**可配置项**：

| 配置项 | 可操作 |
|--------|--------|
| 供应商管理 | 新增 / 删除供应商 |
| 学校管理 | 新增 / 删除学校 |
| 物料二级分类管理 | 按布料/辅料分类分别新增/删除 |
| 尺码选项管理 | 新增 / 删除 |
| 款式选项管理 | 新增 / 删除 |
| 目标车间管理 | 新增 / 删除 |
| 出库目的地管理 | 新增 / 删除 |
| 性别选项管理 | 仅查看，不可修改 |

## 5. 数据看板（老板端增强）

老板「数据总览」支持切换查看各模块图表：

## 7. 通用审批/操作流程图

### 7.1 原材料流转

```
【路径A】出库 → 裁剪管理员（物料指派给裁剪管理员）
原材料入库（原材料/裁剪/车间管理员共同使用同一份原材料库存）
  → 原材料出库（指派给裁剪管理员）
      → 裁剪管理员待裁剪确认（确认/有问题退回）
          → 裁剪加工（填写计划件数）
              → 车间待加工确认（确认/退回）
                  → 加工登记（填写实际件数、辅料使用量）
                      → 成品管理员确认入库
                          → 成品出库

【路径B】出库 → 车间管理员（物料指派给车间管理员）
原材料入库（原材料/裁剪/车间管理员共同使用同一份原材料库存）
  → 原材料出库（指派给车间管理员）
      → 车间管理员待辅料确认（确认/有问题退回）
          → 加工登记（填写实际件数、辅料使用量）
              → 成品管理员确认入库
                  → 成品出库
```

## 8. UI/UX 设计规范

### 8.1 视觉原则

- **字体大小**：正文不小于 32rpx，按钮文字不小于 34rpx
- **触控区域**：所有可点击元素高度不小于 80rpx
- **颜色对比度**：文字与背景对比度满足 WCAG AA 标准
- **层级结构**：最多 3 级页面深度，底部 Tab 不超过 5 个
- **配色方案**：
  - 主色调：深蓝色（`#1A73E8`），传递专业、可靠感
  - 辅助色：浅灰背景（`#F5F5F5`），白色卡片
  - 状态色：成功绿（`#34A853`）、警告橙（`#FB8C00`）、危险红（`#EA4335`）
  - 老板端可额外使用金色（`#F9AB00`）作为点缀，区分管理视角

### 8.2 页面结构规范

- **列表页**：「搜索栏 + 筛选 Tab + 列表卡片 + 底部操作」
- **详情页**：「页面标题 + 基本信息块 + 明细块 + 底部操作按钮」
- **表单页**：「页面标题 + 表单项（上往下排列）+ 底部固定提交按钮」
- **弹窗**：仅用于二次确认、选项新增、原因填写等简单交互，不做复杂表单弹窗

### 8.3 空状态设计

每个列表页在无数据时显示统一空状态插图 + 文字提示，例如：
- 入库记录为空：「暂无入库记录，赶快去新增吧~」

## 12. 附录

### 12.1 术语表

| 术语 | 定义 |
|------|------|
| 一级分类 | 物料大类，本产品固定为「布料」「辅料」 |
| 二级分类 | 物料细分，如布料下可设"棉布""涤纶"等，辅料下可设"纽扣""拉链"等 |
| 损耗率 | 裁剪/加工过程中不可避免的材料损耗比例 |
| SKU | 库存量单位，本产品定义为「性别+款式+学校+尺码」的单一组合 |
| 裁剪单 | 由裁剪管理员创建，记录来料信息和裁剪计划的单据 |
| 加工单 | 由车间管理员创建，记录实际加工数量和辅料消耗的单据 |

### 12.2 待决策项

以下问题需在开发启动前与业务方确认：

1. **库存警戒值**：原材料和成品库存低于多少件时触发预警通知？
2. **损耗率阈值**：损耗率超过多少需要老板审批或特殊标记？
3. **订单删除规则**：已确认/已加工的订单是否允许删除，还是只能作废？
4. **历史数据迁移**：是否有现有 Excel 数据需要导入系统？
5. **多老板意见冲突**：当多个老板对同一配置项有不同意见时的处理机制？
6. **打印需求**：是否需要支持打印入库单/出库单等凭证？
