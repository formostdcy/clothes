# 云函数部署指南

## 为什么我无法直接帮你部署

微信开发者工具的「上传云函数」必须在 GUI 工具里完成，因为它需要：
1. 登录你的微信开发者账号
2. 关联小程序的 AppID
3. 选择具体的云开发环境
4. 跟腾讯云服务器建立连接

这些操作**命令行 / 脚本都做不到**。所以上传必须你手动操作。

## 已经帮你做好的准备

✅ 49 个云函数目录名已与 `package.json` 的 `name` 一致
✅ 49 个 `index.js` 顶部都加了 `const cloud = require('wx-server-sdk')`
✅ 49 个 `package.json` 都加了 `wx-server-sdk` 依赖
✅ 49 个 `wx.cloud.database()` 已替换为 `cloud.database()`
✅ `project.config.json` 已配置 `cloudfunctionRoot` 和 `cloudfunctionTemplateRoot`

## 你需要做的 6 步

### 第 1 步：关闭并重新打开微信开发者工具
（重要！让工具重新识别云函数目录）

### 第 2 步：选择正确的云开发环境
工具顶部「云开发」按钮 → 确认环境 ID 是 `cloud1-d1gyhaxtu1321e4be`

### 第 3 步：在文件树中找到 `cloudfunctions` 文件夹
应该看到里面有 `auth`、`boss`、`cutting`、`employee`、`finished`、
`notification`、`option`、`raw`、`supplier`、`workshop` 共 10 个模块文件夹

### 第 4 步：批量上传所有云函数（推荐）
- **右键点击 `cloudfunctions` 整个文件夹**
- 选择「**同步云函数列表**」→ 等待工具把云函数目录识别为云函数
- 然后**右键点击 `cloudfunctions` 文件夹**
- 选择「**上传并部署：云端安装依赖（不上传 node_modules）**」
- 等待 2~5 分钟（首次安装依赖较慢）

### 第 5 步：逐个上传（如果上面整批上传失败）
- 逐个右键 `cloudfunctions/auth/auth-login` 等
- 选择「**上传并部署：云端安装依赖**」

### 第 6 步：验证部署成功
- 工具顶部「云开发」→「云函数」标签
- 应该能看到 49 个云函数都显示「运行中」或「部署成功」

## 部署后怎么测试

1. 在微信开发者工具里点击「编译」
2. 用真机或模拟器扫码测试登录
3. 如果某个功能报「云函数不存在」错误
   - 检查你上传的环境 ID 跟 `app.js` 里的 `cloud1-d1gyhaxtu1321e4be` 是否一致
   - 在云开发控制台 → 云函数 → 确认该函数已经上传

## 常见报错

| 报错 | 原因 | 解决 |
| --- | --- | --- |
| `ResourceNotFound.Function` | 该云函数没上传 | 按第 5 步逐个上传 |
| `wx-server-sdk 找不到` | 没勾选「云端安装依赖」 | 重新上传时勾上 |
| `FunctionName parameter could not be found` | 函数没上传成功 | 删除后重新上传 |
